# 楽天ROOM自動投稿システム 実装計画 (Implementation Plan)

## 1. プロジェクト概要
楽天ROOMへの自動投稿による「完全放置型の収益化システム」を構築する。
トレンドや特定高収益ジャンルに合わせた商品選定、無料API（Gemini 2.0 Flash）による高品質な紹介文生成、適切なタイミングでの自動投稿をすべて無人で行う。

## 2. システム要件

### 2.1 投稿スケジュール
ユーザーの活動・購買意欲が高まる「ゴールデンタイム」にフォーカスし、以下の1日5回のスケジュールで実行する。
- 0:00 （ゼロコレ対策・日付変更直後の購買層向け）
- 9:00 （朝の家事が一段落した主婦層向け）
- 12:00 （昼休みの会社員向け）
- 21:00 （夜のピークタイム）
- 23:00 （就寝前のピークタイム）
※ GitHub Actionsのcron（UTC）で設定する。

### 2.2 収益化ロジックと商品選定
- **データソース**: 楽天 Developers API (Rakuten Item Search API / Ichiba Item Ranking API)
- **ジャンル特化の切り替え**: 
  - 総合ランキングだけでなく、環境変数設定により特定の「高成約率ジャンル（例：ふるさと納税、季節家電、1000円ポッキリ）」をターゲットとして切り替え可能にする。
- **フィルタリング条件**:
  - 価格: 10,000円以内（衝動買いされやすい価格帯）、またはジャンルごとの適正価格。
  - 在庫ステータス: 「売り切れ」「販売停止」の商品を確実に除外（販売可能フラグの厳格なバリデーション）。
- **特売情報の優先抽出**:
  - APIレスポンスから「ポイントアップ情報」や「クーポン情報」が含まれている商品を検知した場合、優先的にピックアップして紹介文でアピールするロジックを組み込む。

### 2.3 AIライティング（プロンプト設計とコスト最適化）
- **利用API**: Google Gemini 2.0 Flash (Free Tier)
- **コスト・制限対策**: 
  - 無料枠（Free Tier）での利用を前提とする。RPM（Requests Per Minute）やRPD（Requests Per Day）の制約を考慮し、バッチ処理内で適切なスリープ（ウェイト）を入れるリトライ＆レート制限ロジックを実装する。
- **役割とプロンプト方針**:
  - 商品名・商品説明・特売情報・価格から、SNSに適した紹介文を生成する。
  - スペックの羅列ではなく、**ベネフィット（その商品を買うことで得られる体験・メリット）**を強調。
  - ポイントアップやクーポンなどの情報を最優先でアピールし、お得感を強調。
  - 適切なハッシュタグの自動生成（#楽天ROOM #買ってよかった 等）。

### 2.4 実行インフラ・ログイン安定化・通知
- **実行環境**: GitHub Actions を利用したヘッドレス実行環境
- **自動操作ツール**: Playwright (TypeScript)
- **Cookie管理とエラー検知**:
  - セッション情報（Cookie）をGitHub Secretsに保存し、Playwright実行時に注入してログイン状態を完全維持。
  - 実行開始直後にCookieの有効性を検証し、期限切れやCAPTCHA遭遇を「早期検知」するロジックを導入。
- **通知機能**:
  - 投稿の失敗、またはCookieの期限切れ（再ログイン要求）を検知した際、DiscordのWebhook（またはLINE Notify/Slack）を通じて即座にスマホへ通知を送信。
- **ローカル補助ツール**:
  - PCで手動ログインし、最新のCookie文字列を1クリックで取得・フォーマットして出力する `tools/cookie-exporter.ts` を用意し、Cookie更新の手間を最小化する。

## 3. 推奨アーキテクチャ・モジュール構成

プロジェクトディレクトリ構成案:
```
rakuten-room-auto-system/
├── src/
│   ├── fetcher.ts    # 楽天API連携、トレンド・ジャンル抽出、ポイント/クーポン検知
│   ├── generator.ts  # Gemini API (Free Tier) を用いたPR文生成（レート制限対応）
│   ├── poster.ts     # PlaywrightによるROOM投稿（Cookie注入含む）
│   ├── session.ts    # Cookieのパース・管理、有効性検証ユーティリティ
│   ├── notifiers.ts  # Discord/LINE/Slack等へのエラー・ステータス通知
│   └── main.ts       # 全体のオーケストレーション
├── tools/
│   └── cookie-exporter.ts # ローカル用：ブラウザを開きCookieを取得・エクスポートするツール
├── .github/
│   └── workflows/
│       └── auto-post.yml # 定期実行ワークフロー
├── package.json
├── tsconfig.json
└── .env.example
```

## 4. エラーハンドリング方針
- **APIレート制限**: Gemini APIの無料枠制限(429エラー等)や楽天APIのエラー時は、リトライ（Exponential Backoff）とスリープを実施。
- **Cookie期限切れ / CAPTCHA発生**:
  - Playwright実行初期または途中でログイン要求画面へリダイレクトされた場合、速やかにプロセスを停止する。
  - `src/notifiers.ts` を経由し、Discord等のWebhookへ「Cookie更新の必要性」を警告通知として送信する。
- **DOM変更検知**: 楽天ROOM側のUI変更による要素探索のタイムアウト時には、ログ出力とスクリーンショット取得（GitHub Actions Artifact保存）、およびエラー通知を行う。

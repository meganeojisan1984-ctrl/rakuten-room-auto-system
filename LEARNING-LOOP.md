# 自己学習ループエージェント構成

投稿→実績計測→分析→戦略更新→翌日の投稿に反映、を毎日自動で回し、**世代を重ねるごとに売上が伸びる方向へ自己調整**するシステム。

## 部隊編成

```
                 🎖 司令官 (commander.ts)
        監視・戦略決定・デイリーレポート(Discord)
       ┌──────────┬──────────┼──────────┬──────────┐
   🔍 商品選定    ✍ コメント   📮 投稿    📣 SNS拡散   📊 計測→分析
   (scout)     (copywriter)  (poster)  (promoter)  (metrics/analyst)
   fetcher.ts   generator.ts  poster.ts  sns.ts     agents/*.ts
```

| エージェント | 役割 | 学習の受け取り方 |
|---|---|---|
| **司令官** | 全報告をレビューし戦略(strategy.json)を次世代へ更新。**マーチャンダイザーとして季節先取り・高単価・ニーズの型を毎晩考える**。異常検知でDiscord警告 | LLM判断+ルールベースのフォールバック |
| **商品選定 (scout)** | 売上期待値スコア(レビュー実績×お得情報×**期待報酬額=価格×料率**)で商品選定 | ジャンル重み・**価格帯重み(高単価枠)**・**季節キーワード(投稿の約3割)**で選択が変わる |
| **コメント (copywriter)** | 投稿文生成 | 勝ちパターン(styleHints)がプロンプトに注入される |
| **投稿 (poster)** | 楽天ROOMへPlaywright投稿 | 投稿タイプ重み(postTypeWeights)で配分が変わる |
| **SNS拡散 (promoter)** | Instagram/Threadsクロス投稿 | — |
| **計測 (metrics)** | 自ROOMのいいね数をスクレイピングし履歴に記録 | — |
| **分析 (analyst)** | ジャンル/タイプ/時間帯別に実績集計+エージェント健康診断 | — |

## OODAループ × マーケティング・ミックス(4P)

このシステムは **OODAループ** で回る:

| OODA | 担当 | 内容 |
|---|---|---|
| **Observe (観測)** | 計測エージェント | 自ROOMのいいね数を毎晩収集 |
| **Orient (状況判断)** | 分析エージェント | **4Pの軸**で実績を集計・整形 |
| **Decide (意思決定)** | 司令官 | 4Pごとに戦略(重み)を更新 |
| **Act (実行)** | scout/copywriter/poster/promoter | 新戦略で翌日の投稿を実行 |

司令官は **4P** の枠組みで毎晩判断する:

| 4P | 学習対象 | 戦略パラメータ |
|---|---|---|
| **Product** (何を) | ジャンル別実績+季節先取り | genreWeights / seasonalKeywords |
| **Price** (いくらの) | 価格帯別実績×期待報酬 | priceBandWeights |
| **Place** (どこで・いつ) | 時間帯別実績 | notesに傾向を蓄積 |
| **Promotion** (どう魅せる) | 投稿タイプ×フック別実績 | postTypeWeights / hookWeights / styleHints |

## 脱ワンパターンの仕組み（Promotion）

- **8種のフック(書き出しパターン)**: 驚き/共感/問いかけ/数字/ビフォーアフター/損失回避/ストーリー/権威 — 投稿ごとに重み付きローテーション。直近5投稿で使ったフックは選択確率1/4に低下（連続防止）
- **書き出しの重複禁止**: 直近5投稿の書き出し25文字をプロンプトに渡し「似た書き出しNG」と明示
- **フックの学習**: どのフックがいいねを取れるかを司令官が学習(hookWeights)。ただし多様性維持のため2フックまでしか1.5以上にしない
- **SNSのCTAローテーション**: Instagram(CTA5種×タグ3セット)・Threads(誘導文4種)を毎回ランダムに切替

## 学習サイクル（毎日）

1. **8〜23時**: 投稿ワークフロー(auto-post.yml)が5回実行。各エージェントが実行報告を `agent_reports.json` に、投稿内容を `post_history.json` に記録
2. **23:45**: 学習ワークフロー(auto-learn.yml)が起動
   - 計測: 自ROOMプロフィールからいいね数を収集→履歴に反映
   - 分析: ジャンル・投稿タイプ・時間帯・**価格帯**別の平均いいねを集計
   - 司令官: 実績をLLMで分析し `strategy.json` を第N+1世代へ更新、Discordへレポート
     - **売れる仕組みの思考**: ①季節先取り(翌月需要のキーワード5個を更新) ②高単価戦略(価格帯重み。5000円以上は少ないいいねでも期待報酬で勝ちうる) ③ニーズの型(消耗品リピート/季節イベント/買い替え家電)のバランス
3. **翌日**: 投稿エージェント群が新戦略を読んで動く（伸びるジャンルが選ばれやすくなり、勝ちパターンの文体で書く）

## 司令官の監視機能

- エージェントの失敗率50%以上(直近3回以上) → 🚨 個別警告
- Cookie期限切れの兆候 → 🚨 ROOM_COOKIE更新の要請
- 投稿があるのに計測0件 → 🚨 計測エージェント機能不全の警告
- 毎晩のデイリーレポートに全エージェントの成功/失敗数を記載

## 暴走防止の設計

- 重みは必ず **0.5〜2.0にクランプ**（1ジャンルに全振りしない）
- LLMが不通・異常応答の場合は**ルールベース調整**（上位+20%/最下位-10%/平均回帰）に自動フォールバック
- styleHintsは最大3件・各60文字（プロンプト汚染防止）
- 学習ファイルはすべてGitにコミットされるため、`git revert`でいつでも前世代へ戻せる

## 必要な設定

既存のSecrets(ROOM_COOKIE/GROQ_API_KEY/DISCORD_WEBHOOK_URL)に加えて:
- `ROOM_PROFILE_URL`: 自分の楽天ROOMプロフィールURL（いいね計測に必須）

## 手動実行

```bash
npm run learn        # 学習ループをローカルで1回実行
```

GitHub上: Actions → 「自己学習ループ (司令官)」 → Run workflow
